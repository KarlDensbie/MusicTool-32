Klasse: ds_Ton

 

ABGELEITET VON: bcl_Basisdata

Diese Klasse ist ein Datensatz für die Datenbank DB_Töne
Der Datensatz kann von allen Klassen ausgewertet werden. Er kann auch über die Basisklasse bcl_Basisdata angesprochen werden.
Als Typ wird der Typ Tondaten gespeichert.

Aufruf: ds_Ton ( name , index , grundTon , enhU , enhO )


öffentliche Schnittstellen:

    Eingabe:
    • Konstruktor( name , id , grundTon , enhU , enhO )
        • EINZIGE Möglichkeit zur Speicherung der Objektdaten:
        Die Parameter
        • name ,
        • id (als INT) und
        • typ = dType.Tondaten
        werden an die Basisklasse bcl_Basisdata übergeben.
      • Folgende Daten werden gespeichert:
        1. name
          ==> Name des Tons
          ==> TYP: STRING
        2. id
          ==> ID des Tons in der Datenbank DB_Töne
          ==> TYP: eTon
        3. grundTon
          ==> Grundton des Tons (Tonfamilie)
          ==> TYP: eGrundton
        4. enhU
          ==> ID der unteren enharmonischen Verwechselung
          ==> TYP: eTon
        5. enhO
          ==> ID der oberen enharmonischen Verwechselung
          ==> TYP: eTon

    -------------------------------------------------------------

    Ausgabe:
    • Eigenschaften:
      1. TonName
        gibt den Namen des Tons aus
        ==> TYP: STRING
      2. TonID
        gibt den Ton-ID als eTon aus
        ==> TYP: eTon
      3. Grundton
        gibt den Grundton als eGrundton aus
        ==> TYP: eGrundton
      4. IntGrundton
        gibt den Grundton als INT aus
        ==> TYP: INT
      5. EnhU
        gibt den Ton-ID der unteren enharmonischen Verwechselung als eTon aus
        ==> TYP: eTon
      6. IntEnhU
        gibt den Ton-ID der unteren enharmonischen Verwechselung als INT aus
        ==> TYP: INT
      7. EnhO
        gibt den Ton-ID der oberen enharmonischen Verwechselung als eTon aus
        ==> TYP: eTon
      8. IntEnhO
        gibt den Ton-ID der oberen enharmonischen Verwechselung als INT aus
        ==> TYP: INT

      9. geerbte Eigenschaften:
        1. Typ
          gibt den Typ des Datensatzes aus
          ==> TYP: dType
        2. ID
          gibt die ID der Instanz aus
          ==> TYP: INT
        3. Name
          gibt den Namen des Objekts aus
          ==> TYP: STRING

    • Methoden:
      1. enhU()
        gibt die untere enharmonische Verwechselung als Ton-Objekt aus
        Parameter: KEINE
        Rückgabe:
          Ton-Objekt
          ==> TYP: ds_Ton

      2. enhO()
        gibt die obere enharmonische Verwechselung als Ton-Objekt aus
        Parameter: KEINE
        Rückgabe:
          Ton-Objekt
          ==> TYP: ds_Ton
